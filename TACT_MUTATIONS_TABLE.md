# Таблица мутаций Tact

Этот файл — рабочая сводка по итоговому `universalmutator/static/tact.rules`.
В основной ruleset перенесены правила из `tact1.rules`, при этом дубли rule-строк убраны, а уникальные legacy-мутации из старого `tact.rules` сохранены.

## Эффективный ruleset

- стандартный запуск для Tact: `universal.rules` + `tact.rules` + `c_like.rules`
- точечный запуск только Tact-правил: `--only universalmutator/static/tact.rules`
- regex-движок: Python `re`
- Comby-слой для Tact существует отдельно: `universalmutator/comby/tact.rules`

## Проверка покрытия

После слияния `tact1.rules` в основной `tact.rules` проверка по секциям показала, что все мутации из финального файла отражены в таблице.
Отдельные пробельные варианты одних и тех же правил, например для `while (...)` и `while(...)`, сведены в одну строку таблицы и считаются покрытыми.

## Что ещё приходит из общих правил

При обычном запуске поверх `tact.rules` всё ещё подключаются:

- `universal.rules` — базовые мутации арифметики, сравнений, строковых и числовых литералов, `&&/||`, `++/--`, swap аргументов и др.;
- `c_like.rules` — C-like мутации по `if`, `while`, `else`, а также comment skipping для `//`.

Поэтому фактическое покрытие при обычном `mutate foo.tact tact ...` шире, чем один только `static/tact.rules`.

## Что пока не покрыто точечными Tact-правилами

Пробелы всё ещё есть, но они уже заметно уже, чем в старой версии таблицы:

- более аккуратные мутации `SendParameters { ... }` с зависимостями между `value`, `mode`, `bounce`, `code`, `data`;
- безопасные мутации `!!`, различающие optional unwrap и иные контексты;
- более точные storage/collection-мутации для `map<K, V>`, arrays и nested optionals;
- отдельные getter-формы вида `get(<expr>)`, а не только `get fun`.

## Что лучше делать не чистым regex-ом

Слишком хрупко для `static/tact.rules` без AST или хотя бы Comby:

- глубокая переработка `receive(...)` / `external(...)` / `bounced(...)` по сигнатуре сообщений;
- мутации вложенных `SendParameters { ... }` и `message(...)` object literal-ов;
- перестановка или удаление полей в `message` / `struct`;
- мутации generic-типов `map<K, V>` и сложных nested type forms;
- структурные преобразования control-flow и многострочных guard-блоков.
